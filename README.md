# Claude Code Manager

Графическая утилита на PySide6 для запуска **Claude Code** через локальный прокси **Omniroute** или через внешний **Base URL** (FreeModel / inferall.ai / собственный).

---

## Возможности

- Запуск Claude Code из единого окна с автоустановкой переменных окружения.
- Два режима, переключаемых ползунком в шапке:
  - **Omniroute** — локальный прокси `http://localhost:20128/v1` со своим API ключом.
  - **BaseURL** — кастомный шлюз (например, `cc.freemodel.dev`) со своим API ключом и моделью.
- **Жёсткий пин версии Claude Code на `2.1.178`** — устанавливается через `npm install -g @anthropic-ai/claude-code@2.1.178`. Версии выше `2.1.178` блокируют сторонние Base URL и API ключи на стороне Anthropic, поэтому приложение **не запустит** CLI, если установлена более новая версия — вместо запуска появится окно с предложением откатиться.
- **Установка / откат / удаление Claude Code прямо из приложения** — две кнопки в шапке: **Установить Claude Code** (она же откатывает с более новой версии) и **Удалить Claude Code** (с подтверждением). PowerShell для уничтожения залоченных файлов запускается автоматически.
- **Выбор Reasoning Effort** прямо рядом с выбором модели: `low / medium / high / xhigh / max`. Эффорт пишется в три места одновременно (`--effort` флаг CLI, переменная окружения `CLAUDE_CODE_EFFORT_LEVEL`, поле `effortLevel` в `~/.claude/settings.json`) — это обходит известный баг с потерей значения `max`.
- **Автодетект внешней установки** — если ты в фоне через `npm install` поставил другую версию Claude Code, приложение увидит это за 3 секунды (без перезапуска) и обновит индикатор/кнопки.
- **Автофикс `installMethod`** — при старте и после установки приложение чинит `~/.claude.json`, меняя `installMethod: "native"` (остаток install.ps1) на `"global"`. Это убирает надоедливое предупреждение `claude command at C:\Users\<you>\.local\bin\claude.exe missing or broken · run claude install to repair`.
- **Цветовая индикация состояния Claude Code** в шапке: зелёный (установлен и совпадает с пином) / красный (не установлен) / жёлто-оранжевый (установлена не та версия) / нейтральный (версия проверяется).
- Управление списком моделей (для Omniroute).
- Управление списком Base URL (для BaseURL-режима): добавление/удаление, два URL вшиты как дефолтные и удалить их нельзя.
- API ключи после сохранения становятся read-only (как в Omniroute), редактирование — через кнопку **Изменить**.
- Консоль с цветным выводом и проверка статуса Omniroute в реальном времени.
- Автосохранение всех настроек.

## Требования

- **Windows** (PowerShell используется для запуска Claude Code в отдельном окне и для управления npm-пакетом).
- **Node.js + npm** — нужны для установки/удаления Claude Code. На Windows `npm` — это `npm.cmd`, приложение корректно находит его через `PATHEXT`. Если Node.js не установлен, приложение **само предложит поставить его одним кликом через `winget`** — см. раздел «Автоустановка Node.js» ниже.
- Для Omniroute-режима — запущенный локальный сервис Omniroute на `http://localhost:20128`.

## Использование

### Шапка приложения

В верхней части — широкий ползунок с двумя позициями:

- **BaseURL** (слева, оранжевый) — режим внешнего шлюза.
- **Omniroute** (справа, синий) — режим локального прокси.

При переключении неиспользуемые секции полностью скрываются, окно плавно меняет высоту.

Под ползунком — две кнопки управления Claude Code CLI:

- **Установить Claude Code** / **Откатить до v2.1.178** / **Claude Code установлен** — состояние и действие меняются автоматически в зависимости от того, что найдено в системе.
- **Удалить Claude Code** — полностью сносит CLI (с подтверждением).

Ниже — рамка `Claude Code` с цветным индикатором и подписью о текущем состоянии (`Установлен v2.1.178`, `Не установлен`, `Установлена v2.1.200 — нужна v2.1.178`, `Проверка версии…`).

### Автоустановка Node.js

Claude Code ставится через `npm`, а `npm` идёт в комплекте с Node.js. Если в системе нет ни того, ни другого, приложение через ~0.4 секунды после запуска покажет окно **«Нужен Node.js (npm)»**. Дальше работает по двум сценариям:

- **Есть `winget`** (встроен в Windows 10 1809+ и Windows 11) — в окне будет кнопка **«Установить Node.js»**. Клик → откроется окно PowerShell, в котором запустится команда:
  ```powershell
  winget install --id OpenJS.NodeJS.LTS -e --accept-source-agreements --accept-package-agreements
  ```
  Это официальный пакет в репозитории winget, скачивающий MSI прямо с `nodejs.org`. После окончания установки PowerShell ждёт нажатия клавиши и сам напоминает: «Перезапусти Claude Code Manager, чтобы он подхватил npm в PATH».
- **Нет `winget`** (очень старые Windows 10 без обновлений) — кнопка превращается в **«Скачать Node.js»** и открывает в браузере `https://nodejs.org/en/download`. Скачай Windows Installer (.msi) LTS, поставь его и перезапусти приложение.

То же окно появляется, если ты нажмёшь **«Установить Claude Code»** в шапке, а npm не найден — кнопка действия там одинаковая (`winget` → автоустановка, без `winget` → открытие страницы).

### Установка / откат / удаление Claude Code

Приложение работает только с фиксированной версией CLI — **`@anthropic-ai/claude-code@2.1.178`**. Все три кнопки управляются через npm:

- **Не установлен** (красный индикатор) — кнопка `Установить Claude Code`. Клик → подтверждение → откроется окно PowerShell:
  ```powershell
  npm install -g @anthropic-ai/claude-code@2.1.178
  ```
- **Установлена не та версия** (жёлто-оранжевый индикатор) — кнопка `Откатить до v2.1.178`. Клик → подтверждение → npm переустанавливает пакет на нужную версию. Настройки в `%USERPROFILE%\.claude` (история, проекты, конфиг) **не пострадают**.
- **Установлен и совпадает** (зелёный индикатор) — кнопка неактивна.
- **Удалить Claude Code** — открывает диалог подтверждения. Клик → PowerShell прибивает запущенные `claude.exe`, делает `npm uninstall -g @anthropic-ai/claude-code`, и в случае EBUSY (файл залочен) делает повтор через `Remove-Item -Recurse -Force`. Дополнительно чистит остатки от install.ps1 в `~/.local/bin\claude*` и `~/.claude\local\`.

### Почему версия запинена

Anthropic в версиях выше `2.1.178` заблокировала использование сторонних Base URL и API ключей — все запросы уходят только в официальный сервис Anthropic. Поэтому через FreeModel / Omniroute / любые прокси новые версии CLI работать не будут. Если у тебя в системе установлен CLI выше `2.1.178` — при попытке нажать **Запустить Claude Code** появится блокирующее окно с предложением откатиться до `2.1.178`.

### Reasoning Effort

Селектор эффорта расположен в блоке выбора модели и принимает пять значений: `low → medium → high → xhigh → max` с градиентом от зелёного к красному (чем выше — тем дороже). Значение применяется тремя способами одновременно при запуске Claude Code:

1. Аргумент CLI: `claude … --effort <level>` (перебивает всё остальное).
2. Переменная окружения: `CLAUDE_CODE_EFFORT_LEVEL=<level>`.
3. Поле `effortLevel` в `~/.claude/settings.json`.

Тройное дублирование нужно потому, что у `settings.json` есть баг (#65651): значение `max` иногда даунгрейдится до `high`. Флаг CLI это обходит.

### Автофикс installMethod (фикс предупреждения `~/.local/bin/claude.exe missing or broken`)

Раньше Claude Code можно было поставить через `irm https://claude.ai/install.ps1 | iex`, и тогда в `~/.claude.json` записывалось `"installMethod": "native"` + создавался shim в `~/.local/bin\claude.exe`. После того, как ты переходишь на npm-версию или удаляешь файл shim'а вручную, CLI продолжает искать его по этому пути и при каждом запуске пишет:

```
‼ claude command at C:\Users\<you>\.local\bin\claude.exe missing or broken · run claude install to repair
```

Приложение тихо меняет `installMethod` на `"global"` (то есть «я установлен через npm») в двух точках:

- при старте приложения;
- после успешной установки/переустановки через приложение.

После этого предупреждение пропадает.

### Режим Omniroute

1. Запусти локальный сервис Omniroute (порт `20128`).
2. В блоке **Omniroute** жми **Запустить Omniroute** (если ещё не работает). Статус-индикатор станет зелёным.
3. В блоке **Claude Code**:
   - Выбери модель из списка. Кнопки **Добавить модель** / **Удалить модель** позволяют управлять списком.
   - Выбери Reasoning Effort.
   - Вставь API ключ Omniroute → жми **Сохранить**. После сохранения поле блокируется, кнопка превращается в **Изменить**.
   - Выбери директорию (кнопка **Обзор**) — там Claude Code запустится.
4. Жми **Запустить Claude Code** — откроется новое окно PowerShell с активным `claude`.

### Режим BaseURL (FreeModel / inferall.ai / свой)

1. Переключи ползунок влево на **BaseURL**.
2. **Base URL** — выпадающий список. По умолчанию доступен:
   - `https://cc.freemodel.dev`
   Чтобы добавить свой — жми **Управление** → **Добавить**. Дефолтные URL удалить нельзя.
3. **API ключ** — вставь ключ, соответствующий выбранному Base URL (см. предупреждение ниже), жми **Сохранить**. После этого поле блокируется, кнопка превращается в **Изменить**.
4. **Модель** — кастомный список с карточками выбора: **Fable 5**, Opus 4.8, Opus 4.7, Opus 4.6, Sonnet 4.6. Выбор сохраняется в `~/.claude/settings.json` и применяется при следующем запуске Claude Code. При выборе **Fable 5** появляется предупреждение о высокой стоимости модели (см. ниже).
5. Выбери **Reasoning Effort** (`low / medium / high / xhigh / max`).
6. Выбери рабочую директорию.
7. Нажми **Запустить Claude Code**.

### Первый запуск — активация через /logout

Если ты когда-либо логинился в Claude Code через Anthropic (открывался браузер), на диске сохранён файл авторизации `%USERPROFILE%\.claude\.credentials.json`. Он **перекрывает** переменные окружения и Claude Code игнорирует твой API ключ.

Чтобы исправить — **один раз** введи в открытом окне Claude Code:
```
/logout
```
Затем закрой окно и нажми **Запустить Claude Code** снова.

> **Это нужно сделать только один раз.** После этого приложение автоматически подставляет API ключ и Base URL при каждом запуске. Даже если ты поменяешь ключ в настройках — повторно вводить `/logout` не нужно.

Если Claude Code никогда не открывался через Anthropic — `credentials.json` не существует, и `/logout` не нужен вообще.

## ⚠️ Важно: Fable 5 — модель высшего класса

**Claude Fable 5** (`claude-fable-5`) — самая мощная общедоступная модель Anthropic, выпущенная 09 июня 2026.

- Один запрос при `/effort High` может потребовать **до 15% дневного лимита токенов**.
- Fable 5 на `/effort Medium` превосходит Opus 4.8 на xHigh / Max примерно на **5%**.
- Стоимость: **$10 / $50 за млн токенов** (вход / выход) — в 2× дороже Opus 4.8.

При выборе Fable 5 в приложении автоматически появляется предупреждение с напоминанием об этих ограничениях.

---

## ⚠️ Важно: API ключ привязан к Base URL

Каждый API ключ выпускается конкретным провайдером и работает **только** с его Base URL.

- Ключ от `cc.freemodel.dev` **не подойдёт** к примеру `api.inferall.ai` и наоборот.
- Ключ от Omniroute не работает в BaseURL-режиме (там нужен ключ провайдера, а не локальный токен).
- Если при запуске Claude Code ты видишь `401 Unauthorized` / `invalid api key` — значит, выбранная пара (Base URL ↔ API ключ) не совпадает.

**Как избежать ошибки:**

1. Зайди на сайт нужного провайдера (FreeModel, и т. д.), создай ключ именно там.
2. В приложении выбери в выпадающем списке **тот же** Base URL, для которого выпущен ключ.
3. Сохрани ключ. Если переключаешь Base URL — обязательно нажми **Изменить** и вставь ключ от **нового** URL.

Приложение хранит **один** активный `custom_api_key`. Если у тебя несколько ключей от разных провайдеров — придётся перевставлять при переключении.

## Где что хранится

- **Настройки приложения** — `%APPDATA%\ClaudeManager\settings.json`
  Поля: `selected_model`, `models[]`, `auth_token` (Omniroute), `use_custom_token`, `custom_base_url`, `custom_base_urls[]`, `custom_api_key`, `custom_model`, `reasoning_effort`, `working_directory`.
- **Настройки Claude Code CLI** — `~/.claude/settings.json` (поля `model`, `effortLevel` — пишутся приложением перед запуском).
- **Регистрационный конфиг Claude Code** — `~/.claude.json` (приложение чинит здесь `installMethod` с `native` на `global`).

## Решение проблем

| Симптом | Причина / решение |
|---|---|
| Кнопка **Запустить Claude Code** заблокирована | В Omniroute-режиме — сервис ещё не отвечает по `localhost:20128`. Запусти Omniroute и подожди, пока индикатор позеленеет. В BaseURL-режиме кнопка доступна сразу после сохранения ключа. |
| При попытке запуска появляется окно **«Запуск заблокирован»** | У тебя установлена версия выше `2.1.178`. В этих версиях Anthropic заблокировала сторонние Base URL и API ключи. Нажми «Откатить до v2.1.178» в окне — npm переустановит CLI на нужную версию. |
| В окне Claude Code: `401 / invalid api key` | API ключ не подходит к выбранному Base URL — см. раздел "Важно" выше. |
| В окне Claude Code: `Could not connect` или `ECONNREFUSED` к `localhost:20128` | Omniroute не запущен. Жми **Запустить Omniroute** в верхнем блоке. |
| Claude Code открылся, но использует не тот эффорт | Открой `~/.claude/settings.json` — поле `effortLevel` должно совпадать с выбранным в приложении. Если совпадает, но CLI всё равно показывает `high` вместо `max` — это известный баг CLI (#65651), приложение его обходит через флаг `--effort` (он перебивает settings.json). |
| Claude Code открылся, но использует не ту модель | Зайди в окно Claude Code, набери `/model` — увидишь текущую. Для BaseURL-режима модель пишется в `~/.claude/settings.json` при сохранении ключа. Для Omniroute — модель передаётся через `ANTHROPIC_MODEL`. |
| Поле API ключа не редактируется | Это правильно — после сохранения оно блокируется. Жми **Изменить**, чтобы разблокировать. |
| Окно «Нужен Node.js (npm)» при старте | Node.js не установлен. Нажми **«Установить Node.js»** — приложение поставит его через `winget` автоматически. Если winget недоступен, кнопка станет **«Скачать Node.js»** и откроет официальную страницу. После установки **перезапусти приложение**, чтобы оно увидело npm в PATH. |
| «Нужен npm/Node.js, но он у меня есть» | На Windows `npm` — это `npm.cmd`. Если приложение всё равно не видит npm — открой обычный `cmd` и проверь `npm --version`. Если там тоже не работает — добавь папку с Node.js в системный PATH (обычно `C:\Program Files\nodejs`) и перезапусти приложение. |
| `npm error code EBUSY` при удалении | Один или несколько процессов `claude.exe` залочили файл. Приложение само их прибивает и делает повтор; если всё равно не получилось — закрой все терминалы с Claude Code и попробуй ещё раз. |
| ‼ `claude command at C:\Users\<you>\.local\bin\claude.exe missing or broken` | Это старый shim от `install.ps1`. Приложение чинит `installMethod` в `~/.claude.json` при старте (`native` → `global`) — перезапусти приложение один раз, и предупреждение пропадёт. Если файл `~/.local/bin\claude.exe` ещё лежит — можно снести его руками. |
| Ползунок переключился, но настройки не сохранились | Настройки сохраняются автоматически при изменении. Если поведение странное — проверь `%APPDATA%\ClaudeManager\settings.json`, в крайнем случае удали этот файл — приложение создаст новый. |
| Команда `claude` не найдена при запуске | Claude Code CLI не установлен. Жми **Установить Claude Code** в шапке приложения — npm поставит его в `%APPDATA%\npm\`, который должен быть в PATH (он туда добавляется npm'ом сам). |
| Индикатор показывает зелёный, а версия на самом деле не та | Поллер проверяет бинарь раз в 3 секунды по подписи (путь + mtime + size). Если поставил версию через nestandartный способ, который не меняет mtime — перезапусти приложение, при старте оно перечитает версию явно через `claude --version`. |

## Лицензия

Приватная утилита, без публичной лицензии.
