# Claude Code Manager

Графическая утилита на PySide6 для запуска **Claude Code** через локальный прокси **Omniroute** или через внешний **Base URL** (FreeModel и совместимые).

---

## Что это и зачем (простыми словами)

**Claude Code Manager — это одно окно с кнопками, которое делает всё это за тебя.** Никаких ручных правок конфигов и команд в терминале: открыл приложение, выбрал что нужно, нажал «Запустить».

**Что приложение берёт на себя:**

- **Установка в один клик.** Нет Node.js — поставит его через `winget`. Нет Claude Code — поставит подходящую версию через npm. Стоит не та версия — переустановит одной кнопкой.
- **Два режима работы, ты выбираешь.** По умолчанию у нового пользователя включена **«Поддержка новых версий»** — Claude Code обновляется официальным установщиком Anthropic через `claude.ai/install.ps1`, всегда самая свежая версия. Если тебе нужна воспроизводимая проверенная сборка (или ты работаешь через FreeModel / Omniroute) — выключаешь тумблер в шапке и переходишь в **безопасный режим**: приложение фиксирует Claude Code на версии `2.1.173`, на которой всё протестировано, и запрещает CLI молча обновиться.
- **Настройка ключей и сервиса — через поля, а не файлы.** Выбираешь режим (локальный прокси Omniroute или внешний Base URL), заводишь один или несколько API-ключей в окне «Управление», выбираешь модель и «силу размышления» (reasoning effort) — приложение само пропишет это в нужные конфиги Claude Code перед запуском.
- **Управление множеством API-ключей.** В окне «Управление» можно завести сколько угодно именованных ключей, включать/выключать их по одному, а через 5 часов активности приложение просит подтвердить работоспособность (чтобы ты явно проверял, что ключ ещё живой). Активный ключ подсвечен зелёным, выключенный — красным, «пора подтвердить» — жёлтым.
- **Запуск без подводных камней.** Сам обходит блокировку PowerShell, сам чинит типовые поломки конфигов кнопкой **Fix Claude**, сам ставит статус-строку (модель · контекст · сессия · git) одной кнопкой.
- **Видно, что происходит.** Цветной индикатор показывает состояние Claude Code, а бейдж `freemodel.dev` — живой статус сервиса; по клику открывается окно со статистикой в реальном времени.
- **Интерфейс на русском и английском.** Тумблер `RU / EN` в шапке — переключает язык всего приложения на лету, включая заголовки диалогов и текст обновлений.

**Итог:** то, что обычно требует получаса возни с терминалом, документацией и конфигами, здесь сводится к нескольким кликам. Приложение распространяется как готовый `.exe` — ставить Python или зависимости не нужно.

## Возможности

- Запуск Claude Code из единого окна с автоустановкой переменных окружения.
- Два режима, переключаемых ползунком в шапке:
  - **Omniroute** — локальный прокси `http://localhost:20128/v1` со своим API ключом.
  - **BaseURL** — кастомный шлюз (например, `cc.freemodel.dev`) со своим API ключом и моделью.
- **Тумблер «Поддержка новых версий» в шапке.** Определяет, как приложение обращается с версиями CLI. **По умолчанию у нового пользователя тумблер включён** — Claude Code обновляется официальным установщиком Anthropic:
  - блокировка запуска по версии снимается (CLI можно ставить любую);
  - `DISABLE_UPDATES` и `autoUpdates: false` не пишутся;
  - кнопка **Установить Claude Code** переключается на официальный установщик `irm https://claude.ai/install.ps1 | iex` — установка/обновление идёт напрямую с claude.ai;
  - Fix Claude тоже не пишет `DISABLE_UPDATES` обратно.
- **Безопасный режим.** Если выключить тумблер, приложение фиксирует Claude Code на проверенной версии `2.1.173` и глушит встроенный автообновлятор CLI (`DISABLE_UPDATES=1` в `~/.claude/settings.json`, `autoUpdates: false` в `~/.claude.json`). Смысл — гарантировать воспроизводимое поведение: приложение целиком тестируется именно на этой версии, а в связке с FreeModel / Omniroute / любыми сторонними Base URL безопасный режим убирает риск, что CLI за спиной обновится и сломает совместимость. Тумблер запоминается между запусками; переключаться можно в любой момент — без перезапуска приложения.
- **Установка / переустановка / удаление Claude Code прямо из приложения** — две кнопки в шапке: **Установить Claude Code** и **Удалить Claude Code** (с подтверждением). В безопасном режиме кнопка ставит запинованную версию через npm, в режиме автообновлений — запускает официальный установщик. PowerShell для уничтожения залоченных файлов запускается автоматически.
- **Установка / переустановка / удаление status line одной кнопкой** — третья кнопка в шапке **Status line**. Открывает единое окно «Внимание» с превью того, как будет выглядеть строка внизу окна Claude Code (модель · контекст · session ID · git-репозиторий), и тремя кнопками: **Отмена**, **Удалить** (активна только если status line уже стоит) и **Установить / Переустановить**. Перед самим действием — мини-подтверждение «Вы действительно хотите…». Прогресс показывается отдельным окном с процент-баром.
- **Выбор Reasoning Effort** прямо рядом с выбором модели: `low / medium / high / xhigh / max`. Эффорт пишется в три места одновременно (`--effort` флаг CLI, переменная окружения `CLAUDE_CODE_EFFORT_LEVEL`, поле `effortLevel` в `~/.claude/settings.json`) — это обходит известный баг с потерей значения `max`.
- **Переключатель контекста 200K / 1M** — маленький ползунок внутри поля выбора модели. Если включить `1M`, к идентификатору модели добавляется суффикс `[1M]` (работает для Opus 4.8/4.7/4.6, Sonnet 5, Sonnet 4.6, Fable 5).
- **Управление API ключами через отдельное окно.** Кнопка **Управление** рядом с полем ключа открывает широкое окно, где можно завести любое число именованных ключей, включать/выключать каждый ползунком OFF/ON и выбирать активный кликом по карточке. Цвет рамки карточки показывает состояние: зелёный — активен, красный — выключен, жёлтый — прошло 5 часов с активации и приложение просит подтвердить работоспособность (клик по ON сбрасывает таймер и снова красит в зелёный). В окне видно до 4 карточек одновременно, дальше — скролл. Поле ключа на главном экране становится read-only и просто показывает активный ключ; всё редактирование — только в этом окне.
- **Автодетект внешней установки** — если ты в фоне через `npm install` или официальный установщик поставил другую версию Claude Code, приложение увидит это за 3 секунды (без перезапуска) и обновит индикатор/кнопки.
- **Автофикс `installMethod`** — при старте и после установки приложение чинит `~/.claude.json`, меняя `installMethod: "native"` (остаток от `install.ps1`) на `"global"`. Это убирает надоедливое предупреждение `claude command at C:\Users\<you>\.local\bin\claude.exe missing or broken · run claude install to repair`.
- **Автостражи безопасного режима.** В безопасном режиме при каждом запуске приложения два фоновых потока проверяют, что `env.DISABLE_UPDATES=1` стоит в `~/.claude/settings.json` и `autoUpdates: false` — в `~/.claude.json`. Если чего-то нет — мержат тихо. Идемпотентно: если флаги уже стоят, диск не трогают. В режиме «Поддержка новых версий» эти стражи не запускаются.
- **Кнопка Fix Claude** — лечит самый частый сценарий «Claude Code не отвечает / выдаёт ошибки API / Both apiKeyHelper and ANTHROPIC_API_KEY set» после смены способа авторизации. Файл `~/.claude.json` уносится в `.bak` (с уникальным суффиксом, ничего не удаляется), создаётся свежий с `installMethod=global`. `~/.claude/settings.json` **пересоздаётся с нуля** — все «зависшие» ключи от старых установок (`apiKeyHelper`, `hooks`, `mcpServers`, мусор от ручных правок) удаляются. Настройки приложения (модель, эффорт, токен) хранятся отдельно и не теряются — приложение само пропишет их обратно при следующем взаимодействии. Fix уважает выбранный режим: в безопасном режиме дописывает `DISABLE_UPDATES` и `autoUpdates: false`, в режиме автообновлений — не пишет их вовсе.
- **Цветовая индикация состояния Claude Code** в шапке: зелёный (установлен и совпадает с пином) / красный (не установлен) / жёлто-оранжевый (в безопасном режиме — установлена не та версия; в режиме автообновлений — нейтральный текст «Официальные обновления включены (v...)»).
- **Бейдж `freemodel.dev` в левом верхнем углу** — виден только в режиме BaseURL при выбранном `cc.freemodel.dev`. Цветная точка справа от текста отображает реальный статус сервиса (опрос `/api/status`). Клик по бейджу открывает окно статистики freemodel.dev в реальном времени (латентность моделей, гистограмма последних проб, процент успеха). Окно открывается в **установленном браузере** (Chrome/Edge) в режиме приложения — для плавных анимаций и точного рендера; данные ему отдаёт встроенный локальный мини-сервер `127.0.0.1` (поэтому работает независимо от внешнего зеркала и без проблем с CORS). Запускается в изолированном профиле с `--disable-extensions` — расширения вроде Dark Reader не перекрашивают окно. Под бейджем — серая изогнутая стрелка с подписью «клик» как подсказка, что бейдж кликабельный.
- **Авто-bypass PowerShell ExecutionPolicy при запуске Claude Code** — частая проблема у новых пользователей: PowerShell со стандартной политикой `Restricted` блокирует запуск `claude.ps1` с ошибкой `Невозможно загрузить файл … claude.ps1, так как выполнение сценариев отключено`. Приложение автоматически префиксит каждый запуск Claude командой `Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process -Force`, которая действует только внутри запускаемого PowerShell-процесса и **не меняет глобальную политику системы**. На корпоративных машинах с GPO, форсящим `Restricted` через MachinePolicy/UserPolicy, обход не сработает (приоритет GPO выше).
- **Двуязычный интерфейс RU / EN.** Тумблер в шапке переключает язык всего приложения на лету — заголовки диалогов, кнопки, сообщения консоли, окно обновления. Язык запоминается между запусками.
- **Автообновление самого приложения.** При старте проверяется наличие новой версии Claude Code Manager; при её наличии показывается диалог «Доступно обновление!». После подтверждения — окно скачивания с процент-баром, автоматически удаляет старый `.exe` и запускает новый. Все тексты диалога переведены на выбранный язык.
- Управление списком моделей (для Omniroute) через кнопки **Добавить модель** / **Удалить модель**.
- Управление списком Base URL (для BaseURL-режима): добавление/удаление, дефолтные URL защищены от удаления. Если удалить текущий выбранный URL — приложение автоматически переключится на дефолтный `cc.freemodel.dev` и корректно покажет бейдж сервиса (без необходимости перезапуска).
- Консоль с цветным выводом и проверка статуса Omniroute в реальном времени.
- Автосохранение всех настроек.

## Требования

- **Windows** (PowerShell используется для запуска Claude Code в отдельном окне и для управления npm-пакетом).
- **Node.js + npm** — нужны для установки/удаления Claude Code в безопасном режиме. На Windows `npm` — это `npm.cmd`, приложение корректно находит его через `PATHEXT`. Если Node.js не установлен, приложение **само предложит поставить его одним кликом через `winget`** — см. раздел «Автоустановка Node.js» ниже. В режиме «Поддержка новых версий» Node.js не обязателен: официальный установщик Anthropic ставит standalone-версию.
- Для Omniroute-режима — запущенный локальный сервис Omniroute на `http://localhost:20128`.

## Использование

### Шапка приложения

В верхней части — широкий ползунок с двумя позициями:

- **BaseURL** (слева, оранжевый) — режим внешнего шлюза.
- **Omniroute** (справа, синий) — режим локального прокси.

Справа от заголовка «Claude Code» — тумблер **«Поддержка новых версий»** (по умолчанию выключен) и переключатель языка **RU / EN**. При переключении режима BaseURL/Omniroute неиспользуемые секции полностью скрываются, окно плавно меняет высоту.

Под ползунком — четыре кнопки управления Claude Code CLI:

- **Установить Claude Code** / **Установлен ✓ (v...)** — состояние и действие меняются автоматически. В безопасном режиме кнопка предлагает установить/переустановить проверенную версию через npm; в режиме автообновлений — запускает официальный установщик.
- **Удалить Claude Code** — полностью сносит CLI (с подтверждением).
- **Status line** — управление status line в нижней строке окна Claude Code (см. раздел ниже).
- **Fix Claude** — лечение проблем «API не отвечает / ошибки при первом запуске» через перенос `~/.claude.json` в `.bak` и установку правильных стартовых флагов (см. раздел ниже).

Ниже — рамка `Claude Code` с цветным индикатором и подписью о текущем состоянии.

### Безопасный режим и «Поддержка новых версий»

Прямо в шапке рядом с заголовком есть тумблер **«Поддержка новых версий Claude Code»**. Он определяет, как приложение обращается с версиями CLI и автообновлениями.

**Безопасный режим (по умолчанию, тумблер выключен):**

- Claude Code фиксирован на проверенной версии `2.1.173`. Именно на ней приложение протестировано целиком.
- В `~/.claude/settings.json` дописывается `env.DISABLE_UPDATES=1`, в `~/.claude.json` — `autoUpdates: false`. Оба флага мержатся при каждом старте приложения — CLI не сможет обновиться сам.
- Кнопка **Установить Claude Code** ставит именно `@anthropic-ai/claude-code@2.1.173` через npm.
- Если в системе установлена версия новее, при нажатии **Запустить Claude Code** появляется блокирующее окно с предложением откатить (одна кнопка — приложение всё сделает).
- Fix Claude при исправлении конфигов дописывает те же флаги обратно, чтобы после починки безопасный режим сохранялся.

**Режим «Поддержка новых версий» (тумблер включён):**

- При включении показывается предупреждение: приложение снимает защиту `DISABLE_UPDATES`, перестаёт следить за версией и переключает установку на официальный установщик Anthropic. Клик по «Перейти в безопасный режим» отменяет включение.
- После подтверждения:
  - блокировка запуска по версии снимается — CLI может быть любым;
  - `env.DISABLE_UPDATES` удаляется из `~/.claude/settings.json`, `autoUpdates` из `~/.claude.json` тоже уходит;
  - кнопка **Установить Claude Code** меняется на «Обновить Claude Code (официально)» и запускает `irm https://claude.ai/install.ps1 | iex` в отдельном окне PowerShell;
  - индикатор в рамке `Claude Code` показывает нейтральный текст «Официальные обновления включены (v...)» независимо от версии;
  - Fix Claude в этом режиме не пишет `DISABLE_UPDATES` / `autoUpdates` — не возвращает пользователя обратно в безопасный режим.
- Выключение тумблера возвращает всё как было: `DISABLE_UPDATES` и `autoUpdates: false` применяются сразу, без перезапуска, кнопка установки возвращается к npm-варианту.

**Что выбирать:**

- Если ты используешь **FreeModel / Omniroute / любые сторонние Base URL** — оставляй безопасный режим. Он гарантирует, что связка «приложение ↔ CLI ↔ сервис» останется той, на которой всё проверено.
- Если тебе нужны **самые свежие версии Claude Code напрямую от Anthropic** и ты готов к тому, что что-то может отличаться от протестированного, — включай «Поддержка новых версий».

### Автоустановка Node.js

Claude Code в безопасном режиме ставится через `npm`, а `npm` идёт в комплекте с Node.js. Если в системе нет ни того, ни другого, приложение через ~0.4 секунды после запуска покажет окно **«Нужен Node.js (npm)»**. Дальше работает по двум сценариям:

- **Есть `winget`** (встроен в Windows 10 1809+ и Windows 11) — в окне будет кнопка **«Установить Node.js»**. Клик → откроется окно PowerShell, в котором запустится команда:
  ```powershell
  winget install --id OpenJS.NodeJS.LTS --exact --accept-source-agreements --accept-package-agreements
  ```
  После установки Node.js закрой окно PowerShell и **перезапусти приложение** — оно увидит `npm` в PATH.
- **Winget недоступен** (Windows 10 до 1809 или сборки без App Installer) — кнопка станет **«Скачать Node.js»** и откроет официальную страницу [nodejs.org/en/download](https://nodejs.org/en/download). Устанавливаешь как обычную программу, перезапускаешь приложение.

### Установка / удаление Claude Code (безопасный режим)

В безопасном режиме приложение работает только с фиксированной версией CLI — **`@anthropic-ai/claude-code@2.1.173`**. Все три кнопки управляются через npm:

- **Не установлен** (красный индикатор) — кнопка `Установить Claude Code`. Клик → подтверждение → откроется окно PowerShell:
  ```powershell
  npm install -g @anthropic-ai/claude-code@2.1.173
  ```
- **Установлена не та версия** (жёлто-оранжевый индикатор) — кнопка `Откатить до v2.1.173`. Клик → подтверждение → npm переустанавливает пакет на нужную версию. Настройки в `%USERPROFILE%\.claude` (история, проекты, конфиг) **не пострадают**.
- **Установлен и совпадает** (зелёный индикатор) — кнопка неактивна.
- **Удалить Claude Code** — открывает диалог подтверждения. Клик → PowerShell прибивает запущенные `claude.exe`, делает `npm uninstall -g @anthropic-ai/claude-code`, и в случае EBUSY (файл залочен) делает повтор через `Remove-Item -Recurse -Force`. Дополнительно чистит остатки от install.ps1 в `~/.local/bin\claude*` и `~/.claude\local\`.

### Установка / обновление Claude Code (режим автообновлений)

Если включён тумблер **«Поддержка новых версий»**, кнопка **Установить Claude Code** превращается в **«Обновить Claude Code (официально)»**. Клик → подтверждение → откроется окно PowerShell с командой:

```powershell
irm https://claude.ai/install.ps1 | iex
```

Это официальный установщик Anthropic — он скачивает и ставит самую свежую версию CLI в `%USERPROFILE%\.claude\local\` и добавляет её в PATH. Никакого пина, никаких `DISABLE_UPDATES` — обычный запуск, обычные обновления. Настройки в `%USERPROFILE%\.claude` (история, проекты) сохраняются.

Кнопка **Удалить Claude Code** работает независимо от режима — сносит и npm-установку, и standalone.

### Установка / удаление status line

Status line — это нижняя строка в окне Claude Code, в которой видно текущую модель, заполненность контекста, ID сессии и git-репозиторий рабочей папки. Менеджер ставит её одним кликом — без правки `settings.json` руками.

Кнопка в шапке всегда называется **Status line** и открывает одно окно «Внимание» с превью внешнего вида и тремя действиями:

| Состояние | Что нарисовано в окне | Кнопка «Установить / Переустановить» | Кнопка «Удалить» |
|---|---|---|---|
| **Status line не настроен** (свежая система) | Жёлтая строка `● Сейчас status line не настроен`, пояснение что произойдёт, превью | подпись **«Установить»**, активна | неактивна (затемнена) |
| **Status line уже установлен** | Жёлтая строка `● Сейчас status line установлен`, текущая команда в рамке, превью | подпись **«Переустановить»**, активна | активна (красный hover) |

Перед самим действием показывается мини-окно подтверждения «Вы действительно хотите…»:

- **Переустановка** — оранжевая иконка `↻`, текст про то, что текущий блок `statusLine` и файл `~/.claude/statusline-command.sh` будут полностью перезаписаны.
- **Удаление** — красная иконка `×`, текст про то, что блок `statusLine` уйдёт из `~/.claude/settings.json`, а sh-скрипт будет стёрт.
- **Свежая установка** — без второго подтверждения (текст уже был в основном окне).

После подтверждения открывается окно прогресса с процент-баром (8 → 25 → 55 → 80 → 100). Что именно делает приложение:

**Установка / переустановка:**
1. Записывает встроенный в `.exe` скрипт `statusline-command.sh` в `~/.claude/statusline-command.sh`. Запись бинарная, с принудительными LF — Git-Bash на Windows падает на шебанге с CRLF.
2. В `~/.claude/settings.json` записывает блок:
   ```json
   "statusLine": {
     "type": "command",
     "command": "/c/Users/<you>/.claude/statusline-command.sh"
   }
   ```
   Путь конвертируется из `C:\Users\<you>\.claude\…` в Git-Bash-формат `/c/Users/<you>/.claude/…` — это та же конвенция, что и в стандартном клиенте Claude Code. Остальные ключи `settings.json` (model, effortLevel, theme и т.д.) **не трогаются**.

**Удаление:**
1. Из `~/.claude/settings.json` удаляется ключ `statusLine` (другие ключи остаются как есть).
2. Файл `~/.claude/statusline-command.sh` стирается.

После завершения окно прогресса превращается в `Status line установлен ✓` / `Status line удалён ✓` с кнопкой **Готово**.

### Fix Claude

Используется, если Claude Code:

- **не отвечает / висит на `Retrying`** при первом запросе;
- выдаёт **ошибки API** или предупреждение **`Both apiKeyHelper and ANTHROPIC_API_KEY set · auth may not work as expected`**;
- ведёт себя странно после смены способа авторизации (FreeModel → Omniroute и наоборот, перехода с install.ps1 на npm-версию и т.п.).

Подавляющее большинство этих проблем — следствие «грязных» конфигов от предыдущих установок: OAuth-токены, маркеры native-инсталлера, зависшие `apiKeyHelper` / `hooks` / `mcpServers` от старых попыток настроить Claude вручную. Fix Claude **сбрасывает оба конфига до чистого, минимально-рабочего состояния**.

Кнопка **Fix Claude** в шапке открывает красное окно «Внимание» с описанием проблемы и тремя пунктами действий. Кнопка **«Отмена»** с красным hover, кнопка **«Исправить»** активна только если `~/.claude.json` существует — иначе фиксить нечего.

**Что именно делает кнопка «Исправить»** (в порядке выполнения, всё под одним `threading.Lock` чтобы не конфликтовать с фоновыми стражами):

1. Подбирает свободное имя для бэкапа: сначала `~/.claude.json.bak`, потом `.bak.2`, `.bak.3` и т.д. — **ничего не перезатирает**. Если файла нет — фикс выходит с пометкой «фиксить нечего».
2. `os.rename` переносит проблемный `~/.claude.json` в выбранное `.bak`-имя. Атомарно: либо унесли весь файл, либо ничего не делали.
3. Создаёт свежий `~/.claude.json` со стартовым содержимым. В **безопасном режиме**:
   ```json
   {
     "installMethod": "global",
     "autoUpdates": false
   }
   ```
   В режиме **«Поддержка новых версий»**:
   ```json
   {
     "installMethod": "global"
   }
   ```
4. **Полностью пересоздаёт** `~/.claude/settings.json`. В **безопасном режиме**:
   ```json
   {
     "env": {
       "DISABLE_UPDATES": "1"
     }
   }
   ```
   В режиме **«Поддержка новых версий»** — файл создаётся пустым (`{}`), без `DISABLE_UPDATES`.

   Это **главное отличие от старого поведения** классического Fix: раньше он аккуратно мержил только `env.DISABLE_UPDATES` поверх существующего файла. Теперь он сбрасывает файл полностью — и так уходят все «зависшие» ключи (`apiKeyHelper`, чужие `hooks`, кривой `mcpServers` и т.п.), которые могли молча ломать авторизацию.

   `DISABLE_UPDATES=1` — **официально задокументированный** способ выключить автообновление CLI ([code.claude.com/docs/en/setup](https://code.claude.com/docs/en/setup)). Он жёстче `DISABLE_AUTOUPDATER` — блокирует и фоновую самообновлялку, и ручной `claude update`.

**А как же мои настройки?** Все настройки, которые пишет в `settings.json` само приложение (модель, reasoning effort, токены Omniroute, Base URL), хранятся **отдельно** в `%APPDATA%\ClaudeManager\settings.json` и подхватываются обратно в `~/.claude/settings.json` при следующем взаимодействии (выборе модели, сохранении ключа, запуске Claude). Поэтому пересоздание не приводит к «потере настроек» — пользователь даже не замечает.

Единственное исключение — **status line**: если он был установлен, после Fix его придётся переустановить кнопкой Status line (приложение не реконструирует его автоматически). Sh-скрипт остаётся на диске, нужно только нажать «Установить».

**Защита от потери данных:**

- Оригинальный `~/.claude.json` **не удаляется**, а уходит в `.bak` — если что-то пошло не так, можно переименовать обратно.
- Если на каком-то шаге запись падает (нет прав, антивирус) — в логе появится `warning`, а в success-сообщении честно пишется, что именно записалось.

**После завершения** окно прогресса превращается в **«Claude исправлен ✓»** с кнопкой **«Понятно»** — никакого перезапуска приложения не требуется, изменения применяются мгновенно (Claude CLI читает конфиги при каждом запуске сам).

**Что Fix Claude НЕ решает** — проблемы, которые не лежат в конфиг-файлах:

| Симптом | Чем лечить |
|---|---|
| Невалидный API-ключ (`401 / Invalid API key`) | Перевыпустить ключ у провайдера, ввести заново в окне «Управление» |
| Истёк OAuth-токен (FreeModel) | `claude /logout` в открытом окне Claude, потом запустить заново |
| Нет интернета / VPN режет / геоблок | Сеть |
| Anthropic / Omniroute лежит (5xx) | Подождать |
| Rate limit (`429`) | Подождать |
| Claude CLI не установлен | Кнопка **Установить Claude Code** |
| Антивирус блокирует `claude.exe` | Whitelist в антивирусе |

Грубо, **Fix Claude покрывает ~90% реальных проблем запуска** — оставшиеся 10% это креды, сеть и внешние сервисы.

**Автостражи безопасного режима** — даже если ты не нажал Fix Claude, в безопасном режиме при каждом запуске менеджера два фоновых демон-потока:

- `_ensure_disable_updates_in_settings` — проверяет, что в `~/.claude/settings.json` стоит `env.DISABLE_UPDATES=1`. Если нет — мержит (а не пересоздаёт, чтобы не уничтожить пользовательский `settings.json` без явного клика по Fix). Если уже стоит — диск не трогает.
- `_ensure_auto_updates_false_in_claude_json` — то же самое для `autoUpdates: false` в `~/.claude.json`.

Оба используют общий `threading.Lock` с самим Fix Claude — если пользователь нажмёт кнопку в первые миллисекунды после старта, гонок read-modify-write не будет. В режиме «Поддержка новых версий» эти стражи вообще не запускаются.

### Reasoning Effort

Селектор эффорта расположен в блоке выбора модели и принимает пять значений: `low → medium → high → xhigh → max` с градиентом от зелёного к красному (чем выше — тем дороже). Значение применяется тремя способами одновременно при запуске Claude Code:

1. Аргумент CLI: `claude … --effort <level>` (перебивает всё остальное).
2. Переменная окружения: `CLAUDE_CODE_EFFORT_LEVEL=<level>`.
3. Поле `effortLevel` в `~/.claude/settings.json`.

Тройное дублирование нужно потому, что у `settings.json` есть баг (#65651): значение `max` иногда даунгрейдится до `high`. Флаг CLI это обходит.

### Переключатель 200K / 1M контекст

Внутри поля выбора модели в правом углу — маленький ползунок с двумя значениями: **200K** (слева, синий, дефолт) и **1M** (справа, зелёный). Если включить `1M`, к идентификатору модели при запуске добавляется суффикс `[1M]` — поддерживается для Opus 4.8/4.7/4.6, Sonnet 5, Sonnet 4.6 и Fable 5. При наведении неактивная сторона плавно подсвечивается своим цветом. Выбор запоминается между запусками.

### Автофикс installMethod (фикс предупреждения `~/.local/bin/claude.exe missing or broken`)

Раньше Claude Code можно было поставить через `irm https://claude.ai/install.ps1 | iex`, и тогда в `~/.claude.json` записывалось `"installMethod": "native"` + создавался shim в `~/.local/bin\claude.exe`. После того, как ты переходишь на npm-версию или удаляешь файл shim'а вручную, CLI продолжает искать его по этому пути и при каждом запуске пишет:

```
‼ claude command at C:\Users\<you>\.local\bin\claude.exe missing or broken · run claude install to repair
```

Приложение тихо меняет `installMethod` на `"global"` (то есть «я установлен через npm») в двух точках:

- при старте приложения;
- после успешной установки/переустановки через приложение.

После этого предупреждение пропадает.

### Режим Omniroute

1. Запусти локальный сервис Omniroute (порт `20128`).
2. В блоке **Omniroute** жми **Запустить Omniroute** (если ещё не работает). Статус-индикатор станет зелёным.
3. В блоке **Claude Code**:
   - Выбери модель из списка. Кнопки **Добавить модель** / **Удалить модель** позволяют управлять списком.
   - Выбери Reasoning Effort.
   - Вставь API ключ Omniroute → жми **Сохранить**. После сохранения поле блокируется.
   - Выбери директорию (кнопка **Обзор**) — там Claude Code запустится.
4. Жми **Запустить Claude Code** — откроется новое окно PowerShell с активным `claude`.

### Режим BaseURL (FreeModel и совместимые)

1. Переключи ползунок влево на **BaseURL**.
2. **Base URL** — выпадающий список. По умолчанию доступен `https://cc.freemodel.dev`. Чтобы добавить свой — жми **Управление** → **Добавить**. Дефолтные URL удалить нельзя.
3. **API ключ** — нажми кнопку **Управление** справа от поля ключа, чтобы открыть окно управления ключами (см. следующий раздел). В самом поле на главном экране ключ теперь отображается read-only — там всегда виден активный ключ.
4. **Модель** — кастомный список с карточками выбора: **Fable 5**, Opus 4.8, Opus 4.7, Opus 4.6, Sonnet 5, Sonnet 4.6. Выбор сохраняется в `~/.claude/settings.json` и применяется при следующем запуске Claude Code. При выборе **Fable 5** появляется предупреждение о высокой стоимости модели (см. ниже).
5. Внутри поля выбора модели — маленький ползунок **200K / 1M**: включай `1M`, если тебе нужен расширенный контекст (поддержано на Opus 4.8/4.7/4.6, Sonnet 5, Sonnet 4.6, Fable 5).
6. Выбери **Reasoning Effort** (`low / medium / high / xhigh / max`).
7. Выбери рабочую директорию.
8. Нажми **Запустить Claude Code**.

### Управление API ключами

Кнопка **Управление** справа от поля API ключа открывает широкое окно, в котором происходит всё редактирование ключей. Само поле ключа на главном экране — read-only и просто отображает активный ключ.

**Что можно в окне:**

- **Добавить новый ключ.** Внизу два поля: «Название» и «Ключ», рядом зелёная кнопка **Добавить**. Название можно оставить пустым — тогда ключ получит имя вида `Ключ 3`. Первый заведённый ключ автоматически становится активным.
- **Включить / выключить ключ.** Каждая карточка ключа — с ползунком OFF/ON слева. Клик по стороне ON включает и сбрасывает 5-часовой таймер работоспособности; клик по OFF — выключает.
- **Выбрать активный ключ.** Клик по телу карточки зелёного ключа — он становится «выбранным». Слева вверху карточки появляется точка-индикатор, рамка усиливается. Одновременно выбран один ключ; именно его значение уходит в `ANTHROPIC_API_KEY` при запуске Claude. Красные (выключенные) и жёлтые ключи выбрать нельзя — сначала подтвердить/включить.
- **Показать / скрыть значение.** Кнопка-глаз на карточке; длинный ключ показывается с многоточием в середине, чтобы не разъезжаться по ширине.
- **Удалить ключ.** Крестик справа — с подтверждением (жёлтая иконка «внимание», предупреждение что действие безвозвратно).

**Цветовая кодировка рамки карточки:**

| Цвет | Что значит |
|---|---|
| 🟢 **Зелёный** | Активен, прошло меньше 5 часов с момента активации — ключ считается «свежим», можно смело пользоваться. |
| 🔴 **Красный** | Выключен пользователем через ползунок OFF. В `ANTHROPIC_API_KEY` не попадёт, при запуске игнорируется. |
| 🟡 **Жёлтый** | Прошло **ровно 5 часов** с активации. Приложение просит тебя подтвердить, что ключ ещё работает — клик по стороне ON на ползунке сбрасывает таймер и снова красит карточку в зелёный. |

**Максимум 4 карточки видны одновременно** — если ключей больше, окно снабжается вертикальной прокруткой (окно не растягивается бесконечно). Позиция окна на экране запоминается по левому верхнему углу — при удалении ключей окно только уменьшается вниз, не «прыгает».

**Как это связано с запуском Claude Code:**

При каждом запуске приложение берёт **выбранный зелёный ключ**, кладёт его в `ANTHROPIC_API_KEY` и передаёт CLI. Если выбранный ключ вдруг оказался жёлтым (прошло 5 часов) — используется первый попавшийся зелёный по списку; если зелёных нет вообще — приложение честно пишет в консоль «Нет активного API ключа — включите ключ в окне «Управление» и прерывает запуск.

Все ключи хранятся локально в `%APPDATA%\ClaudeManager\settings.json` в поле `api_keys[]` (id, name, value, enabled, activated_at, selected_key_id).

### Первый запуск — активация через /logout

Если ты когда-либо логинился в Claude Code через Anthropic (открывался браузер), на диске сохранён файл авторизации `%USERPROFILE%\.claude\.credentials.json`. Он **перекрывает** переменные окружения и Claude Code игнорирует твой API ключ.

Чтобы исправить — **один раз** введи в открытом окне Claude Code:
```
/logout
```
Затем закрой окно и нажми **Запустить Claude Code** снова.

> **Это нужно сделать только один раз.** После этого приложение автоматически подставляет API ключ и Base URL при каждом запуске. Даже если ты поменяешь ключ в окне «Управление» — повторно вводить `/logout` не нужно.

Если Claude Code никогда не открывался через Anthropic — `credentials.json` не существует, и `/logout` не нужен вообще.

## ⚠️ Важно: API ключ привязан к Base URL

Каждый API ключ выпускается конкретным провайдером и работает **только** с его Base URL.

- Ключ от `cc.freemodel.dev` **не подойдёт** к другому провайдеру и наоборот.
- Ключ от Omniroute не работает в BaseURL-режиме (там нужен ключ провайдера, а не локальный токен).
- Если при запуске Claude Code ты видишь `401 Unauthorized` / `invalid api key` — значит, выбранная пара (Base URL ↔ API ключ) не совпадает.

**Как избежать ошибки:**

1. Зайди на сайт нужного провайдера (FreeModel и т.д.), создай ключ именно там.
2. В приложении выбери в выпадающем списке **тот же** Base URL, для которого выпущен ключ.
3. В окне **Управление** заведи ключ (или выбери один из уже заведённых, соответствующий этому Base URL).

Приложение поддерживает **несколько ключей одновременно** через окно «Управление» — можно завести ключи от разных провайдеров и переключать активный кликом по карточке, не удаляя ничего.

## Где что хранится

- **Настройки приложения** — `%APPDATA%\ClaudeManager\settings.json`
  Поля: `selected_model`, `models[]`, `auth_token` (Omniroute), `use_custom_token`, `custom_base_url`, `custom_base_urls[]`, `custom_api_key` (зеркало активного ключа для обратной совместимости), `api_keys[]` (список именованных ключей), `selected_key_id`, `custom_model`, `reasoning_effort`, `use_1m_context`, `auto_update_enabled`, `app_language`, `working_directory`.
- **Настройки Claude Code CLI** — `~/.claude/settings.json` (поля `model`, `effortLevel` — пишутся приложением перед запуском; блок `statusLine` — при установке status line через кнопку; блок `env` с `DISABLE_UPDATES=1` — при старте приложения и через Fix Claude в безопасном режиме). **Fix Claude пересоздаёт этот файл с нуля**, оставляя только `env.DISABLE_UPDATES` (в безопасном режиме) или полностью пустым (в режиме автообновлений); всё остальное приложение восстановит самостоятельно при следующем взаимодействии.
- **Скрипт status line** — `~/.claude/statusline-command.sh` (записывается приложением из встроенного в `.exe` скрипта, бинарно с LF).
- **Регистрационный конфиг Claude Code** — `~/.claude.json` (приложение чинит здесь `installMethod` с `native` на `global`; `autoUpdates: false` дописывает только в безопасном режиме).
- **Бэкапы Fix Claude** — `~/.claude.json.bak`, `.bak.2`, `.bak.3`, … (уникальный суффикс на каждый запуск Fix Claude, ничего не перезатирается).

## Решение проблем

| Симптом | Причина / решение |
|---|---|
| Кнопка **Запустить Claude Code** заблокирована | В Omniroute-режиме — сервис ещё не отвечает по `localhost:20128`. Запусти Omniroute и подожди, пока индикатор позеленеет. В BaseURL-режиме кнопка доступна сразу после добавления хотя бы одного зелёного ключа в окне «Управление». |
| При попытке запуска появляется окно **«Запуск заблокирован»** | Ты в безопасном режиме, а установлена версия CLI выше `2.1.173`. Нажми «Откатить до v2.1.173» в окне — npm переустановит CLI на нужную версию. Либо включи тумблер «Поддержка новых версий» в шапке — тогда блокировка снимется. |
| В консоли: **«Нет активного API ключа — включите ключ в окне «Управление»»** | В окне «Управление» нет ни одного зелёного (активного) ключа. Открой окно, добавь новый или включи существующий по ползунку OFF/ON. |
| Карточка ключа стала **жёлтой** | Прошло 5 часов с момента активации — приложение просит подтвердить, что ключ ещё живой. Клик по ON на ползунке карточки сбросит таймер и снова покрасит её в зелёный. |
| В окне Claude Code: `401 / invalid api key` | API ключ не подходит к выбранному Base URL — см. раздел «Важно» выше. Открой окно «Управление», проверь, что выбран правильный ключ. |
| В окне Claude Code: `Could not connect` или `ECONNREFUSED` к `localhost:20128` | Omniroute не запущен. Жми **Запустить Omniroute** в верхнем блоке. |
| Claude Code открылся, но использует не тот эффорт | Открой `~/.claude/settings.json` — поле `effortLevel` должно совпадать с выбранным в приложении. Если совпадает, но CLI всё равно показывает `high` вместо `max` — это известный баг CLI (#65651), приложение его обходит через флаг `--effort` (он перебивает settings.json). |
| Claude Code открылся, но использует не ту модель | Зайди в окно Claude Code, набери `/model` — увидишь текущую. Для BaseURL-режима модель пишется в `~/.claude/settings.json` при смене модели или запуске. Для Omniroute — модель передаётся через `ANTHROPIC_MODEL`. |
| Окно «Нужен Node.js (npm)» при старте | Node.js не установлен. Нажми **«Установить Node.js»** — приложение поставит его через `winget` автоматически. Если winget недоступен, кнопка станет **«Скачать Node.js»** и откроет официальную страницу. После установки **перезапусти приложение**, чтобы оно увидело npm в PATH. Альтернатива — включи «Поддержка новых версий»: тогда Node.js не нужен, ставится standalone-версия. |
| «Нужен npm/Node.js, но он у меня есть» | На Windows `npm` — это `npm.cmd`. Если приложение всё равно не видит npm — открой обычный `cmd` и проверь `npm --version`. Если там тоже не работает — добавь папку с Node.js в системный PATH (обычно `C:\Program Files\nodejs`) и перезапусти приложение. |
| `npm error code EBUSY` при удалении | Один или несколько процессов `claude.exe` залочили файл. Приложение само их прибивает и делает повтор; если всё равно не получилось — закрой все терминалы с Claude Code и попробуй ещё раз. |
| ‼ `claude command at C:\Users\<you>\.local\bin\claude.exe missing or broken` | Это старый shim от `install.ps1`. Приложение чинит `installMethod` в `~/.claude.json` при старте (`native` → `global`) — перезапусти приложение один раз, и предупреждение пропадёт. Если файл `~/.local/bin\claude.exe` ещё лежит — можно снести его руками. |
| Ползунок BaseURL/Omniroute переключился, но настройки не сохранились | Настройки сохраняются автоматически при изменении. Если поведение странное — проверь `%APPDATA%\ClaudeManager\settings.json`, в крайнем случае удали этот файл — приложение создаст новый. |
| Команда `claude` не найдена при запуске | Claude Code CLI не установлен. Жми **Установить Claude Code** в шапке. В безопасном режиме поставится через npm в `%APPDATA%\npm\`, в режиме автообновлений — через официальный установщик в `%USERPROFILE%\.claude\local\`. |
| `claude : Невозможно загрузить файл … claude.ps1, так как выполнение сценариев отключено` (`UnauthorizedAccess` / `PSSecurityException`) | PowerShell блокирует запуск `.ps1` со стандартной политикой `Restricted`. **Приложение лечит это автоматически** — каждый запуск Claude префиксится командой `Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process -Force`, она снимает блок только для этой сессии PowerShell, никаких системных настроек не меняет. Если ошибка всё равно появляется — это значит, что политика залочена через GPO (корпоративные машины): открой PowerShell от админа и выполни `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope LocalMachine -Force` (нужны права администратора). |
| Индикатор показывает зелёный, а версия на самом деле не та | Поллер проверяет бинарь раз в 3 секунды по подписи (путь + mtime + size). Если поставил версию через нестандартный способ, который не меняет mtime — перезапусти приложение, при старте оно перечитает версию явно через `claude --version`. |
| Установил status line, но внизу окна Claude Code ничего не появилось | Проверь `~/.claude/settings.json` — там должен быть блок `"statusLine": { "type": "command", "command": "/c/Users/<you>/.claude/statusline-command.sh" }`. Если есть, но строки нет — закрой и заново открой окно Claude Code (он читает `settings.json` только при запуске). |
| Claude Code не отвечает / ошибки API / `Both apiKeyHelper and ANTHROPIC_API_KEY set` | Нажми **Fix Claude** в шапке. Старый `~/.claude.json` уйдёт в `.bak`, создастся свежий. `~/.claude/settings.json` пересоздастся с нуля — все зависшие ключи (`apiKeyHelper`, чужие `hooks`, мусор) удалятся. Перезапуск приложения не нужен. Если что-то пошло не так — переименуй `.bak` обратно. |
| Хочу вернуться на самую свежую версию Claude Code | Включи тумблер **«Поддержка новых версий»** в шапке рядом с заголовком «Claude Code», подтверди переход, потом нажми **Обновить Claude Code (официально)** — откроется PowerShell с официальным установщиком. |
| Хочу вернуться в безопасный режим | Выключи тумблер **«Поддержка новых версий»** — `DISABLE_UPDATES` и `autoUpdates: false` вернутся сразу, без перезапуска. Дальше нажми **Установить Claude Code** для отката на `2.1.173`, если стоит более новая версия. |
| Невалидный API-ключ (`401 / Invalid API key`) | Fix Claude **не решит** — проблема в самом ключе, а не в конфиге. Перевыпусти ключ у провайдера (FreeModel, Omniroute, …), удали старый в окне «Управление» и добавь новый. |
| Истёк OAuth-токен (FreeModel, видно по `Authentication required`) | Fix Claude **не решит**. Открой окно Claude, напиши `/logout`, потом запусти заново через приложение — авторизация пройдёт заново. |
